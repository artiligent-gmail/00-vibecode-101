# Project Brief: ระบบจัดการทรัพย์สินถาวร (Fixed Asset Management System)

## บทสรุปผู้บริหาร (Executive Summary)
ระบบจัดการทรัพย์สินถาวร (Fixed Asset Management System) เป็นเว็บแอปพลิเคชันที่ออกแบบมาเพื่อปรับปรุงกระบวนการติดตามทรัพย์สินขององค์กรให้ทันสมัย แก้ปัญหาความยุ่งยากและข้อผิดพลาดจากการจัดการด้วย Excel แบบเดิมๆ โดยระบบทำหน้าที่เป็นศูนย์กลางข้อมูลดิจิทัลสำหรับฮาร์ดแวร์และอุปกรณ์ต่างๆ เหมาะสำหรับผู้ดูแลระบบไอทีและผู้จัดการสำนักงานในธุรกิจ SME ระบบมาพร้อมกับหน้าจอใช้งานที่สวยงามและใช้งานง่าย ช่วยให้มองเห็นสถานะและตำแหน่งของทรัพย์สินได้แบบ Real-time ซึ่งจะช่วยลดการสูญหายและเพิ่มความรับผิดชอบในการใช้งาน

## ที่มาและความสำคัญของปัญหา (Problem Statement)
ปัจจุบันหลายองค์กรยังคงประสบปัญหาการติดตามทรัพย์สินที่กระจัดกระจาย โดยส่วนใหญ่พึ่งพาการจดบันทึกใน Excel ซึ่งมักเกิดข้อผิดพลาดได้ง่าย ส่งผลให้เสียเวลาอย่างมากในระหว่างการตรวจสอบทรัพย์สินประจำปี ไม่สามารถระบุตำแหน่งอุปกรณ์ได้ทันที และขาดความรับผิดชอบเมื่อทรัพย์สินเสียหายหรือสูญหาย ในขณะที่โซลูชันระดับองค์กรที่มีขายทั่วไปมักมีราคาแพงและซับซ้อนเกินความจำเป็น จึงมีความต้องการเร่งด่วนสำหรับระบบที่เรียบง่ายแต่มีประสิทธิภาพเพื่อกลับมาควบคุมทรัพย์สินขององค์กรได้อีกครั้ง

## แนวทางการแก้ไขปัญหา (Proposed Solution)
เราได้พัฒนาเว็บแอปพลิเคชันแบบ Monolithic โดยใช้ **Next.js 15 App Router** เพื่อประสิทธิภาพสูงและรองรับ SEO โซลูชันหลักประกอบด้วยระบบจัดการทะเบียนทรัพย์สินที่ครบถ้วน, โมดูลการเบิก-จ่าย (Check-in/Check-out) ที่ใช้งานง่าย, และ Dashboard แสดงผลแบบ Real-time จุดเด่นสำคัญคือการออกแบบ UI ที่ทันสมัยด้วย **Shadcn/Tailwind**, ความถูกต้องของข้อมูลที่เชื่อถือได้ผ่าน **Prisma/PostgreSQL**, และการเน้น Workflow ที่จำเป็นเพื่อลดความยุ่งยากในการใช้งานประจำวัน

## กลุ่มผู้ใช้งานเป้าหมาย (Target Users)
### กลุ่มผู้ใช้งานหลัก: ผู้ดูแลระบบไอที / ผู้จัดการฝ่ายปฏิบัติการ
*   **โปรไฟล์:** ผู้ที่มีความเชี่ยวชาญด้านเทคโนโลยี รับผิดชอบการจัดซื้อและแจกจ่ายอุปกรณ์ฮาร์ดแวร์
*   **พฤติกรรม:** ปัจจุบันจัดการทรัพย์สินผ่าน Excel; รู้สึกหงุดหงิดที่ไม่มีข้อมูลแบบ Real-time
*   **ความต้องการ:** ค้นหาทรัพย์สินได้รวดเร็ว, มอบหมายให้พนักงานได้ง่าย, และรายงานสถานะได้ชัดเจน
*   **เป้าหมาย:** รักษาความถูกต้องของบัญชีทรัพย์สินให้ได้ 100% และลดขั้นตอนการ Onboarding/Offboarding พนักงาน

## เป้าหมายและตัวชี้วัดความสำเร็จ (Goals & Success Metrics)
### วัตถุประสงค์ทางธุรกิจ
*   ติดตามทรัพย์สินได้ถูกต้องแม่นยำ 100%
*   ลดเวลาในการตรวจสอบทรัพย์สิน (Audit) ลง 50%
*   ลดการสูญหายของทรัพย์สินที่เกิดจากการไม่ได้บันทึกการเบิกจ่าย
### ตัวชี้วัดความสำเร็จของผู้ใช้ (User Success Metrics)
*   เวลาในการลงทะเบียนทรัพย์สินใหม่ < 2 นาที
*   เวลาในการทำรายการเบิก-จ่าย < 30 วินาที
### ตัวชี้วัดหลัก (KPIs)
*   **Utilization Rate:** อัตราส่วนทรัพย์สินที่ถูกใช้งานเทียบกับที่มีอยู่ทั้งหมด
*   **Churn Rate:** ความถี่ในการหมุนเวียนทรัพย์สิน (การเบิก-จ่าย)

## ขอบเขตของ MVP (MVP Scope)
### ฟีเจอร์หลัก (ต้องมี)
*   **Dashboard:** ภาพรวมแบบ Real-time ของจำนวนทรัพย์สินทั้งหมด, ว่าง, กำลังใช้งาน, และซ่อมบำรุง; พร้อมประวัติการทำรายการล่าสุด
*   **Asset Management:** จัดการประเภททรัพย์สิน (CRUD) และทะเบียนทรัพย์สิน (Create/Read)
*   **Operations:** ระบบเบิกจ่าย (Check-out) ให้พนักงาน และรับคืน (Check-in)
*   **Employee Integration:** ระบบรายชื่อพนักงานสำหรับเลือกเพื่อมอบหมายทรัพย์สิน (Seeding data)
### สิ่งที่อยู่นอกขอบเขต MVP
*   ระบบยืนยันตัวตนผู้ใช้งาน (Login/Signup)
*   การแก้ไขและลบทรัพย์สิน (Asset Editing/Deletion) - รอการพัฒนา
*   การคำนวณค่าเสื่อมราคาและรายงานทางการเงิน
*   การสแกน Barcode/QR Code
*   การอัปโหลดรูปภาพ
### เกณฑ์ความสำเร็จของ MVP
MVP จะถือว่าประสบความสำเร็จเมื่อผู้ดูแลระบบสามารถดำเนินการวงจรชีวิตของทรัพย์สินได้ครบถ้วน (สร้าง -> มอบหมาย -> รับคืน -> ดูประวัติ) โดยไม่มีข้อผิดพลาด และแสดงผลรายงานบน Dashboard ได้อย่างถูกต้อง

## วิสัยทัศน์หลัง MVP (Post-MVP Vision)
### ฟีเจอร์ในเฟส 2
*   **Authentication:** ระบบล็อกอินที่ปลอดภัยผ่าน NextAuth.js
*   **Advanced Management:** ความสามารถในการแก้ไข/ลบทรัพย์สิน
*   **Search & Filtering:** การค้นหาขั้นสูงและการกรองข้อมูล
### วิสัยทัศน์ระยะยาว
พัฒนาสู่ชุดโปรแกรมบริหารจัดการทรัพย์สินไอที (ITAM) เต็มรูปแบบ ที่เชื่อมต่อกับระบบจัดซื้อ, การแจ้งเตือนอัตโนมัติ, และรองรับแอปพลิเคชันบนมือถือ

## ข้อพิจารณาทางเทคนิค (Technical Considerations)
### ความต้องการของแพลตฟอร์ม
*   **แพลตฟอร์มเป้าหมาย:** เว็บ (ปรับแต่งสำหรับ Desktop, รองรับ Mobile)
*   **เบราว์เซอร์ที่รองรับ:** เบราว์เซอร์สมัยใหม่ (Chrome, Edge, Safari, Firefox)
### เทคโนโลยีที่เลือกใช้
*   **Frontend:** Next.js 15, React, Tailwind CSS, Shadcn UI
*   **Backend:** Next.js Server Actions
*   **Database:** PostgreSQL (Neon), Prisma ORM
### สถาปัตยกรรม
*   **โครงสร้าง:** Monolithic App Router
*   **ความปลอดภัย:** ตรวจสอบข้อมูลด้วย Zod, ตรวจสอบ Logic ฝั่ง Server

## ข้อจำกัดและสมมติฐาน (Constraints & Assumptions)
### ข้อจำกัด
*   **ระยะเวลา:** การส่งมอบ MVP ในทันที
*   **ทรัพยากร:** สภาพแวดล้อมการพัฒนาโดยนักพัฒนาคนเดียว
### สมมติฐานหลัก
*   ผู้ใช้งานมีการเชื่อมต่ออินเทอร์เน็ตที่เสถียรสำหรับเข้าถึงฐานข้อมูลบน Cloud
*   ปริมาณข้อมูลเริ่มต้นสามารถจัดการได้ภายในขีดความสามารถของฐานข้อมูล Relational ทั่วไป

## ความเสี่ยงและคำถามที่ยังเปิดอยู่ (Risks & Open Questions)
### ความเสี่ยงหลัก
*   **ความผิดพลาดจากการป้อนข้อมูล:** การป้อนข้อมูลด้วยมือโดยไม่มีการตรวจสอบ (เช่น Serial Number) อาจนำไปสู่ข้อมูลซ้ำซ้อน (ลดความเสี่ยงด้วย Unique Constraints ในฐานข้อมูล)
### คำถามที่ยังเปิดอยู่
*   ความต้องการเฉพาะสำหรับการจัดการทรัพย์สินสถานะ "ตัดจำหน่าย" (Retired)?
*   ความจำเป็นในการรองรับหลายสถานที่ (Multi-location)?
